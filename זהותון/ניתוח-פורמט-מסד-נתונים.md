## סקירה כללית

מסד הנתונים מאורגן בארכיטקטורת **Columnar Inverted Index** - כל שדה (עמודה) מאוחסן בקובץ נפרד. זוהי ארכיטקטורה דומה במושגים ל-Apache Lucene/Elasticsearch, אך בפורמט בינארי מותאם אישית.

### עקרון הפעולה

```
חיפוש "כהן" בשם משפחה:
                                                    
  משפחה.TXB (אינדקס הפוך)          תז.TYB (מיפוי ראשי)
  ┌──────────────────────┐          ┌──────────────────┐
  │ Index Section:       │          │ Row 0: TZ→seq    │
  │  "כהן" → ptr, count │──┐       │ Row 1: TZ→seq    │
  │  "כהנא" → ptr, count│  │       │ ...               │
  │  ...                 │  │       └──────────────────┘
  ├──────────────────────┤  │
  │ Data Section:        │  │       תז.TLB (reverse lookup)
  │  [row1][row2][row3]  │◄─┘       ┌──────────────────┐
  │  ...                 │          │ seq→TZ            │
  └──────────────────────┘          └──────────────────┘
           │
           ▼
    row IDs → אותם row IDs קיימים בכל שאר קבצי TXB
    כך מאחזרים שם פרטי, כתובת, טלפון וכו'
```

### למה הארכיטקטורה הזו?

1. **חיפוש בזמן O(log n)** - האינדקסים ממוינים → ניתן לחיפוש בינארי
2. **שליפת תוצאות O(1)** - כל row ID מוביל ישירות לנתונים
3. **חיפוש משולב** - Intersection של מערכי row IDs בין שדות שונים
4. **גישה סלקטיבית** - אין צורך לקרוא את כל מסד הנתונים, רק את הקבצים הרלוונטיים

## מבנה קובץ TXB - אינדקס הפוך

זהו סוג הקובץ המרכזי. כל שדה מיוצג על ידי קובץ TXB אחד.

### מבנה כללי

```
┌─────────────────────────────────────────┐
│           Header (1024 bytes)           │  ← metadata פנימי (לא נדרש לקריאה)
├─────────────────────────────────────────┤
│         Index Section (ממוין!)          │  ← רשומות: [ערך][מצביע][מונה]
│  record 0: [value][ptr][count]          │
│  record 1: [value][ptr][count]          │
│  record 2: [value][ptr][count]          │
│  ...                                    │
│  record N: [value][ptr][count]          │
├─────────────────────────────────────────┤
│         Data Section                    │  ← מערכי row IDs (uint32)
│  [rowID][rowID][rowID]...               │  ← 4 bytes per ID
│  [rowID][rowID]...                      │
│  ...                                    │
└─────────────────────────────────────────┘
```

### מבנה רשומת אינדקס

כל רשומת אינדקס מורכבת מ:

```
┌──────────────────────┬──────────────┬──────────────┐
│   Value Field        │   Pointer    │    Count     │
│   (V bytes)          │  (4 bytes)   │  (4 bytes)   │
│   Windows-1255 text  │  uint32 LE   │  uint32 LE   │
│   null-padded        │  file offset │  num of IDs  │
└──────────────────────┴──────────────┴──────────────┘
          Total: V + 8 bytes per record
```

**שדות:**
- **Value Field** - ערך השדה מקודד ב-Windows-1255, מרופד באפסים עד לרוחב קבוע
- **Pointer** - offset (בבתים) מתחילת הקובץ אל מערך ה-row IDs ב-Data Section
- **Count** - מספר ה-row IDs (= כמה אנשים יש עם הערך הזה)

### אלגוריתם קריאה (Pseudocode)

```python
def read_txb_index(filepath, value_field_size):
    """
    קורא את כל ה-index section של קובץ TXB.
    
    Args:
        filepath: נתיב הקובץ
        value_field_size: גודל שדה הערך בבתים (8-24)
    
    Returns:
        dict: מיפוי value → (file_offset, count)
    """
    HEADER_SIZE = 1024
    RECORD_SIZE = value_field_size + 8  # value + ptr(4) + count(4)
    
    with open(filepath, 'rb') as f:
        f.seek(HEADER_SIZE)
        
        # קרא את הרשומה הראשונה כדי למצוא את תחילת ה-Data Section
        first_record = f.read(RECORD_SIZE)
        data_section_start = struct.unpack('<I', first_record[value_field_size:value_field_size+4])[0]
        
        # חשב מספר רשומות
        index_size = data_section_start - HEADER_SIZE
        num_records = index_size // RECORD_SIZE
        
        # קרא את כל האינדקס
        f.seek(HEADER_SIZE)
        index = {}
        for i in range(num_records):
            record = f.read(RECORD_SIZE)
            value = record[:value_field_size].decode('cp1255').rstrip('\x00')
            ptr = struct.unpack('<I', record[value_field_size:value_field_size+4])[0]
            count = struct.unpack('<I', record[value_field_size+4:value_field_size+8])[0]
            index[value] = (ptr, count)
        
        return index


def get_row_ids(filepath, ptr, count):
    """
    קורא מערך row IDs מה-Data Section.
    
    Args:
        ptr: file offset ל-Data Section
        count: מספר row IDs לקרוא
    
    Returns:
        list[int]: רשימת row IDs
    """
    with open(filepath, 'rb') as f:
        f.seek(ptr)
        data = f.read(count * 4)
        return list(struct.unpack(f'<{count}I', data))
```

### חיפוש בינארי באינדקס

מכיוון שהאינדקס **ממוין** בסדר לקסיקוגרפי, ניתן לבצע חיפוש בינארי:

```python
def binary_search_txb(filepath, value_field_size, target):
    """
    חיפוש בינארי בתוך ה-index section.
    מורכבות: O(log n) קריאות דיסק
    """
    HEADER = 1024
    RS = value_field_size + 8
    
    with open(filepath, 'rb') as f:
        # מצא data_section_start מהרשומה הראשונה
        f.seek(HEADER)
        rec = f.read(RS)
        data_start = struct.unpack('<I', rec[value_field_size:value_field_size+4])[0]
        num_records = (data_start - HEADER) // RS
        
        # חיפוש בינארי
        lo, hi = 0, num_records - 1
        target_encoded = target.encode('cp1255')
        
        while lo <= hi:
            mid = (lo + hi) // 2
            f.seek(HEADER + mid * RS)
            rec = f.read(RS)
            value = rec[:value_field_size].rstrip(b'\x00')
            
            if value == target_encoded:
                ptr = struct.unpack('<I', rec[value_field_size:value_field_size+4])[0]
                count = struct.unpack('<I', rec[value_field_size+4:value_field_size+8])[0]
                return (ptr, count)
            elif value < target_encoded:
                lo = mid + 1
            else:
                hi = mid - 1
        
        return None  # לא נמצא
```

## מבנה קובץ TYB - מיפוי ת.ז.

קובץ `תז.TYB` הוא **מיפוי ראשי** ממספר תעודת זהות למספר שורה סדרתי.

### מבנה

```
┌──────────────────────────────────────┐
│         Header (1024 bytes)          │
├──────────────────────────────────────┤
│  Record 0: [TZ][null][seq_idx][null] │  ← 20 bytes per record
│  Record 1: [TZ][null][seq_idx][null] │
│  Record 2: [TZ][null][seq_idx][null] │
│  ...                                 │
│  Record 8,305,635: ...               │
└──────────────────────────────────────┘
```

### מבנה רשומה (20 bytes)

```
Offset  Size  Type     Description
──────  ────  ──────   ──────────────────────
0       9     ASCII    מספר ת.ז. (9 ספרות, מרופד אפסים משמאל)
9       3     null     ריפוד
12      4     uint32   מספר שורה סדרתי (sequential index)
16      4     null     ריפוד
──────────────────────────────────────────────
Total: 20 bytes
```

### דוגמה

```
Row 0: TZ="         " (empty)  seq=2
Row 1: TZ="000000018"          seq=3
Row 2: TZ="000000026"          seq=4
Row 3: TZ="000000042"          seq=5
Row 4: TZ="000000059"          seq=6
...
```

**חשוב:** הקובץ **ממוין לפי ת.ז.** → מאפשר חיפוש בינארי ב-O(log n)!

### סה"כ

- גודל: **166,113,744 bytes**
- רשומות: **(166,113,744 - 1024) / 20 = 8,305,636**
- חיפוש לפי ת.ז.: **~23 צעדי חיפוש בינארי** (log₂ 8.3M)

## מבנה קובץ TLB - טבלת מיקום

קובץ `תז.TLB` הוא **מיפוי הפוך**: ממספר שורה סדרתי ← מספר ת.ז.

### פורמט

```
┌─────────────┐
│ "תז="       │ ← 3 bytes header (CP1255)
│ "0000000   "│ ← 16 bytes: first entry (padded with spaces)
│ "="         │ ← separator
│ "000000018" │ ← 9 bytes: second TZ value
│ "="         │
│ "000000026" │
│ "="         │
│ ...         │
└─────────────┘
```

### מבנה

- **Header:** `"תז="` (3 bytes)
- **Entry 0:** 16 bytes (ערך ריק מרופד ברווחים) + `"="` 
- **Entry 1+:** 9 bytes (ת.ז.) + `"="` (10 bytes per entry)
- **סה"כ entries:** ~8,305,635

### שימוש

מאפשר **Reverse Lookup**: בהינתן מספר שורה סדרתי (שמגיע מה-Data Section של TXB), 
ניתן לחלץ את מספר הת.ז.:

```python
def get_tz_by_seq(tlb_path, seq_index):
    """
    מחזיר מספר ת.ז. לפי מספר שורה סדרתי.
    """
    HEADER = 3        # "תז="
    FIRST_ENTRY = 17  # "XXXXXXXXXXXXXXXX=" (16 chars + =)
    ENTRY_SIZE = 10   # "NNNNNNNNN=" (9 digits + =)
    
    if seq_index == 0:
        offset = HEADER
        read_len = 16
    else:
        offset = HEADER + FIRST_ENTRY + (seq_index - 1) * ENTRY_SIZE
        read_len = 9
    
    with open(tlb_path, 'rb') as f:
        f.seek(offset)
        return f.read(read_len).decode('ascii').strip()
```

## מבנה קובץ TZB - אינדקס משני

קבצי TZB קיימים רק עבור שדות שמות וישובים: `משפחה.TZB`, `פרטי.TZB`, `ישוב.TZB`, `רחוב.TZB`.

### תפקיד

אלו **אינדקסים ללא רווחים** ו/או **חיפוש חלקי** (substring/phonetic) - הערכים בהם הם ללא רווחים ותווים מיוחדים, מה שמאפשר חיפוש flexibler.

### מבנה

המבנה זהה ל-TXB, אבל עם ערכים מנורמלים:

| קובץ | Value bytes | Record bytes | ערכים ייחודיים | דוגמה |
|-------|-------------|-------------|----------------|-------|
| `משפחה.TZB` | 20 | 28 | 229,005 | "אאביאד", "אאביסטה" |
| `פרטי.TZB` | 16 | 24 | 107,407 | "א", "אהרון" |
| `ישוב.TZB` | 12 | 20 | 6,664 | "1199", "3400" |
| `רחוב.TZB` | 16 | 24 | 14,219 | "ימין" |

**הבדלים מ-TXB רגיל:**
- הרווחים מוסרים מהערכים (`"א אחמד"` → `"אאחמד"`)
- מאפשר חיפוש ישיר ללא ניחוש ריווח
- ערכים מנורמלים מאפשרים **fuzzy search** טוב יותר

## זרימת חיפוש - דוגמאות מעשיות

### 1. חיפוש לפי מספר ת.ז.

```
Input: "012345678"
                    
Step 1: Binary search in תז.TYB
        → TZ("012345678") → seq_index = 42
        
Step 2: seq_index = 42 → use as row_id
        Search each TXB file for record containing row_id 42
        
Step 3: For each field (family, first_name, city, etc.):
        - Scan TXB index section to find which value contains row_id 42
        - [Alternative: use TLB + other structures for direct lookup]
```

**אופטימיזציה:** במקום לסרוק כל TXB, ניתן לבנות **reverse index** בזיכרון: `row_id → value` לכל שדה.

### 2. חיפוש לפי שם משפחה

```
Input: "כהן"

Step 1: Binary search in משפחה.TXB index section
        → found: ptr=0x01A23000, count=87,432
        
Step 2: Read 87,432 row IDs from Data Section at offset 0x01A23000
        → [rowID_1, rowID_2, ..., rowID_87432]
        
Step 3: For display, retrieve additional fields for each row:
        - Look up each rowID in תז.TYB → get TZ number
        - OR read from other TXB data sections
```

### 3. חיפוש משולב (שם משפחה + שם פרטי + עיר)

```
Input: family="כהן", first="דוד", city="ירושלים"

Step 1: Search משפחה.TXB → row_ids_family = {r1, r2, ..., r87432}
Step 2: Search פרטי.TXB  → row_ids_first  = {r5, r8, ..., r45000}
Step 3: Search ישוב.TXB  → row_ids_city   = {r3, r7, ..., r150000}

Step 4: Intersection:
        result = row_ids_family ∩ row_ids_first ∩ row_ids_city
        
Step 5: For each rowID in result:
        Look up additional details from other TXB files
```

**שיפור ביצועים:** תמיד להתחיל מהשדה עם הכי מעט תוצאות (selectivity).

### 4. חיפוש לפי טלפון (Reverse Lookup)

```
Input: "02-5321216"

Step 1: Binary search in משפחהלחיפוש.TXB
        → found: ptr=X, count=13
        
Step 2: Read 13 row IDs → these point to family names
        
Step 3: Cross-reference with פרטילחיפוש.TXB and רחובלחיפוש.TXB
```

### המלצה 1: Memory-Mapped Files

```python
import mmap

class TXBReader:
    def __init__(self, filepath, value_field_size):
        self.f = open(filepath, 'rb')
        self.mm = mmap.mmap(self.f.fileno(), 0, access=mmap.ACCESS_READ)
        self.vfs = value_field_size
        self.rs = value_field_size + 8
        
        # Find data section start from first record
        ptr_bytes = self.mm[1024+self.vfs : 1024+self.vfs+4]
        self.data_start = struct.unpack('<I', ptr_bytes)[0]
        self.num_records = (self.data_start - 1024) // self.rs
```

**יתרונות:**
- מערכת ההפעלה מנהלת את ה-cache
- אין צורך לטעון 1.7GB ל-RAM
- גישה אקראית מהירה מאוד
- שיתוף זיכרון בין processes

### המלצה 3: מנגנון חיפוש חכם

```python
class SmartSearchEngine:
    """
    סדר עדיפות חיפוש אופטימלי:
    1. שדות עם selectivity גבוהה → פחות תוצאות → intersection מהיר
    2. שימוש ב-set intersection ליעילות
    """
    
    SELECTIVITY_ORDER = [
        'תז',        # ייחודי - 1 תוצאה
        'טלפון',     # 678K ייחודיים
        'פרטי',      # 490K ייחודיים
        'משפחה',     # 342K ייחודיים
        'רחוב',      # 26K ייחודיים
        'ישוב',      # 1.5K ייחודיים
        'שנה',       # 445 ייחודיים
        'חודש',      # 255 ייחודיים
        'יום',       # 275 ייחודיים
        'מצב אישי',  # 3.5K ייחודיים
    ]
    
    def search(self, **criteria):
        # מיין את השדות לפי selectivity (הכי סלקטיבי קודם)
        sorted_fields = sorted(
            criteria.items(),
            key=lambda x: self.SELECTIVITY_ORDER.index(x[0])
                          if x[0] in self.SELECTIVITY_ORDER else 999
        )
        
        result_set = None
        for field, value in sorted_fields:
            row_ids = self.search_field(field, value)
            if result_set is None:
                result_set = set(row_ids)
            else:
                result_set &= set(row_ids)  # Intersection
            
            # Early termination
            if len(result_set) == 0:
                return []
        
        return result_set
```

### המלצה 4: Autocomplete מהיר

```python
def autocomplete(self, field, prefix, max_results=10):
    """
    prefix search על האינדקס הממוין.
    לדוגמה: prefix="כה" → ["כהן", "כהנא", "כהנוביץ", ...]
    """
    reader = self.readers[field]
    
    # חיפוש בינארי למציאת הגבול התחתון
    lo = reader.lower_bound(prefix)
    
    results = []
    for i in range(lo, min(lo + max_results, reader.num_records)):
        value, _, count = reader.read_record(i)
        if not value.startswith(prefix):
            break
        results.append({'value': value, 'count': count})
    
    return results
```

### המלצה 5: חיפוש Fuzzy דרך TZB

```python
def fuzzy_search(self, field, query):
    """
    1. נרמל את הquery (הסר רווחים, תווים מיוחדים)
    2. חפש ב-TZB (ללא רווחים)
    3. עשה prefix search + Levenshtein distance
    """
    normalized = query.replace(' ', '').replace('-', '').replace("'", '')
    
    # חפש ב-TZB
    tzb_reader = self.tzb_readers[field]
    candidates = tzb_reader.prefix_search(normalized[:3])  # 3 תווים ראשונים
    
    # דירוג לפי Levenshtein distance
    ranked = []
    for candidate in candidates:
        dist = levenshtein(normalized, candidate['value'])
        if dist <= 2:  # סף מרחק
            ranked.append({**candidate, 'distance': dist})
    
    return sorted(ranked, key=lambda x: x['distance'])
```

### המלצה 6: מציאת קשרי משפחה

```python
def find_family(self, tz):
    """
    מוצא את כל בני המשפחה דרך שדות תזאב/תזאם.
    
    1. מצא את ת.ז. ההורים של האדם
    2. חפש את כל מי שיש לו את אותו תזאב/תזאם
    3. בנה עץ משפחתי
    """
    person = self.lookup_by_tz(tz)
    
    # מצא אחים (אותם הורים)
    siblings = []
    if person['תזאב']:
        siblings += self.search_field('תזאב', person['תזאב'])
    if person['תזאם']:
        siblings += self.search_field('תזאם', person['תזאם'])
    
    # מצא ילדים (האדם הוא הורה)
    children_as_father = self.search_field('תזאב', tz)
    children_as_mother = self.search_field('תזאם', tz)
    
    return {
        'person': person,
        'siblings': siblings,
        'children': children_as_father + children_as_mother
    }
```

## סיכום טכני

### נקודות חוזק של הפורמט
- ✅ אינדקסים ממוינים → חיפוש בינארי O(log n)
- ✅ Columnar storage → гישה סלקטיבית
- ✅ Inverted index → חיפוש לפי כל שדה
- ✅ Row IDs אחידים → intersection פשוט
- ✅ אינדקסים משניים (TZB) → חיפוש fuzzy
- ✅ קבצי "לחיפוש" → reverse phone lookup

### אתגרים
- ⚠️ אין פורמט מתועד רשמי
- ⚠️ קבצים בינאריים (לא קריאים ישירות)
- ⚠️ קידוד Windows-1255 (צריך המרה ל-UTF-8 בממשק)
- ⚠️ Value field size שונה בין קבצים (חייבים metadata)
- ⚠️ Header 1024 bytes לא מתועד (אבל לא צריך אותו)
